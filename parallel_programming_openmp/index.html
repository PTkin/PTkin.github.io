并行程序设计(2) OpenMP

Posted on 2016-06-14

这学期上了一门《并行程序设计》，学习了并行程序设计的三种主流方式：分别是 MPI、OpenMP、Pthreads。于是打算写几篇笔记，对这门课学习的内容做一个简单的整理记录。

本文是第二篇，主要包括了 OpenMP 的基础与基本使用方法，也包括了几个简单的算法的串行代码与使用 OpenMP 进行并行化之后的相应源码。由于本人是在 OSX 环境下开发，配置 OpenMP 的时候也查了一下有关 GCC、LLVM、Clang 之间的历史发展区别联系等资料，这部分资料也整理在本文中。

OpenMP 基础

有关 并行程序设计基础 的内容可以回顾我的上一篇博客 并行程序设计(1)MPI ，这里直接开始讲 OpenMP。

OpenMP

针对 共享内存 并行编程的 API。

OpenMP 与 Pthreads 的区别

Pthreads 要求程序员显式地明确每个线程的行为。而 OpenMP 允许程序员只需要简单地声明一些代码应该并行执行，而 由编译器和运行时系统来决定哪个线程具体执行哪个任务。这也意味着 OpenMP 和 Pthreads 还有另一个不同之处，即 Pthreads 是一个能够被链接到 C 程序的函数库，因此只要系统有 Pthreads 库，Pthreads C 程序就能够被任意 C 编译器使用。相反，OpenMP 要求编译器支持某些操作，所以某些编译器是无法编译 OpenMP 程序的。

OSX 下配置 OpenMP

很少遇到在 Windows 上看起来轻而易举能配置成功，但是在 OSX 环境下配置却这么麻烦的情况。

简要解释一下为什么在 OSX 上配置 OpenMP 这么麻烦的原因：

  1. OpenMP 是由编译器和运行时系统来决定哪个线程具体执行哪个任务，所以需要编译器的支持。
  2. 我们在 Windows 和 Linux 上常用的 GCC 编译器默认支持 OpenMP。（本文后面部分会详细介绍 GCC、LLVM、Clang 的区别联系）
  3. 在 OSX 上苹果公司默认使用自己的 LLVM Compiler，而不是 GCC。（PS：在 OSX 的终端输入 gcc -v 得到的也是 LLVM 的版本信息，而不是 gcc 的版本信息）
  4. LLVM Compiler 默认不支持 OpenMP。

所以我们可能有以下几种解决方式：

  1. 安装 GCC

    gcc 支持 OpenMP。

  2. 为 LLVM 安装 OpenMP 支持

    通过安装 clang-omp，这种方式我也尝试了一下，发现有坑，没有配置成功，因此选择了第一种方式。

下面以 安装 GCC 的方式 进行 OpenMP 的配置。

配置

由于已经安装了 homebrew，所以我使用了它来辅助安装。

  1. 安装 gcc 编译器

    brew install gcc

  2. 查看 gcc 版本 (brew list gcc)

    brew list gcc

    得到 gcc 版本号在下一步有用到。

  3. 编译

    gcc-6 -g -Wall -O2 -fopenmp -o pt_hello_omp ./pt_hello_omp.c
    • gcc-6 命令指定使用版本为 6 的 gcc 编译器（我的 gcc 版本是 6.1.0）。如果使用 gcc命令，则会使用 LLVM 编译器。
    • -g 表示允许使用调试器；
    • -Wall 表示显示警告
    • -O2 表示启动编译器优化（原理可网上搜索）
